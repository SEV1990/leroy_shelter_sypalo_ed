# Shelter Leroy — сайт притулку

Сайт волонтерського притулку для евакуйованих тварин (ГО «Захист тварин України»).
Реалізація дизайну з Claude Design: **статичний фронтенд** (HTML/CSS/JS, вигляд 1-в-1
з макетом) + **бекенд** на Node/Express/SQLite з адмінкою, каталогом тварин і формами
заявок зі сповіщенням у Telegram.

## Структура

```
app/
├── frontend/            # статичний сайт (віддається бекендом як статика)
│   ├── index.html       # односторінковий сайт (12 «сторінок»-секцій, UA/EN)
│   ├── css/styles.css   # увесь стиль + анімації з дизайну
│   ├── js/app.js        # навігація, i18n, слайдшоу, чат, адмінка, виклики API
│   └── assets/
│       ├── img/         # фото (тварини, hero, команда, тихий фронт…)
│       └── fonts/       # Manrope + Bebas Neue (self-hosted woff2)
└── backend/             # API + сервер статики
    ├── src/server.js    # Express, маршрути, авторизація, Telegram
    ├── src/db.js        # SQLite, схема, початкове наповнення
    ├── src/seed.js      # стартовий каталог тварин
    ├── .env.example     # зразок конфігурації
    └── uploads/         # завантажені фото тварин (поза git)
```

## Запуск локально

```bash
cd app/backend
npm install
cp .env.example .env        # відредагуйте: ADMIN_PASSWORD, JWT_SECRET
npm start                   # http://localhost:3000
```

Бекенд віддає і фронтенд, і API з одного порту. При першому запуску база
наповнюється 12 тваринами з каталогу (`src/seed.js`).

## Конфігурація (`.env`)

| Змінна | Призначення |
|---|---|
| `PORT` | порт сервера (за замовч. 3000) |
| `ADMIN_PASSWORD` | пароль входу в адмінку |
| `JWT_SECRET` | секрет для підпису токенів адміна |
| `DB_PATH` | (необов.) шлях до файлу SQLite |
| `TELEGRAM_BOT_TOKEN` | (необов.) токен бота для сповіщень |
| `TELEGRAM_CHAT_ID` | (необов.) куди слати сповіщення |

Якщо токен/чат Telegram не задані — заявки просто зберігаються в базу
(сповіщення вимкнені, помилок немає).

## Адмінка

Відкрити: кнопка 🔑 (правий нижній кут) або пункт меню «Адмін» → ввести пароль.
Вкладки:
- **Тварини** — додавання / **редагування** / видалення (фото файлом або URL).
- **Команда** — додавання / редагування / видалення волонтерів (фото, роль, порядок).
- **Схема** — керування вольєрами шелтеру: номер, хто всередині, статус
  (зайнятий / порожній / не функціонує), ширина, блок, порядок. Лічильник унизу
  схеми (зайнято / порожніх / не функціонує) рахується автоматично.
- **Контакти** — редагування контактної інформації (сайт, Instagram, Telegram, адреса).
- **Заявки** — перегляд і видалення заявок із форм контактів та евакуації.

Контент (тварини, команда, схема, контакти) віддається з бази через API, тож сайт
оновлюється одразу після збереження в адмінці — без правок коду.

## API

| Метод | Шлях | Доступ | Опис |
|---|---|---|---|
| POST | `/api/admin/login` | публічно | `{password}` → `{token}` |
| GET | `/api/animals` | публічно | список тварин |
| POST / PUT / DELETE | `/api/animals[/:id]` | адмін | створити / редагувати / видалити (multipart + `photofile`) |
| GET | `/api/team` | публічно | команда |
| POST / PUT / DELETE | `/api/team[/:id]` | адмін | створити / редагувати / видалити (multipart + `photofile`) |
| GET | `/api/enclosures` | публічно | вольєри (схема) |
| POST / PUT / DELETE | `/api/enclosures[/:id]` | адмін | створити / редагувати / видалити |
| GET | `/api/settings` | публічно | контактна інформація (key/value) |
| PUT | `/api/settings` | адмін | оновити налаштування |
| POST | `/api/contact` | публічно | заявка з форми контактів |
| POST | `/api/evacuation` | публічно | запит на евакуацію |
| GET / DELETE | `/api/requests[/:id]` | адмін | список / видалення заявок |

Адмінські запити вимагають заголовок `Authorization: Bearer <token>`.

## Деплой

Будь-який хост, де працює Node 18+ (Railway, Render, VPS):
1. Залийте репозиторій, `npm install` у `app/backend`.
2. Задайте env-змінні (`ADMIN_PASSWORD`, `JWT_SECRET`, за бажання Telegram).
3. `npm start`. Для постійних даних змонтуйте том під `data.sqlite` та `uploads/`
   (або винесіть `DB_PATH` на постійний диск).
